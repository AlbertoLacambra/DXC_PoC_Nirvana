    ðŸ¤– Dify Dataset Configuration - Nirvana Knowledge Portal¶

    Overview¶

    This guide walks you through creating and configuring a Dify Knowledge Dataset for the Nirvana Knowledge Portal.

    Prerequisites¶

    • VPN connected (OpenVPN)
    • Database nirvana_knowledge created
    • Access to Dify UI: http://10.0.2.91

    Step 1: Access Dify Interface¶

    1. Connect to VPN: 

      sudo openvpn --config /mnt/c/PROYECTS/dify-azure-private-deployment/vpn-config/mi-laptop.ovpn

    2. Open Dify in browser:

    3. URL: http://10.0.2.91
    4. Login with your Dify credentials

    Step 2: Create New Knowledge Base¶

    1. Navigate to Knowledge:
    2. Click on "Knowledge" in the left sidebar

    3. Click "+ Create Knowledge" button

    4. Configure Basic Settings: 

      Name: Nirvana Knowledge Portal
Description: Technical documentation, ADRs, code examples, and runbooks for DXC Cloud Mind - Nirvana PoC
Icon: ðŸ§  (brain emoji)

    5. Select Knowledge Type:

    6. Choose: "External Knowledge Base" (we'll use our custom pgvector database)
    7. OR choose: "Internal" if you want Dify to manage everything

    Step 3: Configure Retrieval Settings¶

    Embedding Model Configuration¶

    1. Select Embedding Provider:
    2. Provider: Azure OpenAI
    3. Model: text-embedding-3-large

    4. Dimensions: 3072

    5. Embedding Settings: 

      Batch Size: 100
Max Tokens per Chunk: 800
Overlap Tokens: 100

    Retrieval Configuration¶

    1. Retrieval Method:
    2. Method: Vector Search + Keyword

    3. This combines semantic search (vector) with traditional keyword search

    4. Retrieval Parameters: 

      Top K: 5
Score Threshold: 0.50
Max Tokens: 4000
Rerank: Enabled
Rerank Model: Cohere Rerank (or compatible)

    Note: Score threshold of 0.50 is recommended based on production testing.
    Typical similarity scores range 0.50-0.65 for relevant matches with: - text-embedding-3-large (3072 dimensions) - Multilingual queries (Spanish/English) - Semantic (not exact) matching

    1. Chunking Strategy: 
      Automatic Chunking: Disabled (we do custom chunking)
Chunk Size: 800 tokens
Chunk Overlap: 100 tokens
Separator: "\n\n" (double newline)

    Step 4: Connect to Vector Database (pgvector)¶

    If using External Knowledge Base:

    1. Database Connection: 

      Database Type: PostgreSQL (pgvector)
Host: dify-postgres-9107e36a.postgres.database.azure.com
Port: 5432
Database: nirvana_knowledge
User: difyadmin
Password: [from secret]
SSL Mode: require

    2. Table Mapping: 

      Embeddings Table: knowledge_chunks
Vector Column: embedding
Content Column: content
Metadata Columns:
  - source_type
  - category
  - file_path
  - repository
  - tags
  - language

    Step 5: Configure Indexing Rules¶

    1. File Type Filters: 

      Allowed Extensions:
  - .md (Markdown)
  - .py (Python)
  - .ts (TypeScript)
  - .tsx (TypeScript React)
  - .yaml (YAML)
  - .yml (YAML)
  - .json (JSON)
  - .sh (Shell scripts)

Excluded Patterns:
  - node_modules/**
  - .git/**
  - dist/**
  - build/**
  - *.min.js
  - *.map

    2. Metadata Extraction: 

      Extract from Git:
  - Commit SHA
  - Author
  - Commit Date
  - Branch

Extract from File:
  - File Path
  - File Size
  - Last Modified
  - Language (from extension)

    Step 6: Create Test Documents¶

    Upload a few test documents to verify the setup:

    1. Test Document 1 - Create test-architecture.md: 

      # Architecture Overview

This is a test document to verify the Knowledge Portal setup.

## Components
- AKS Cluster: dify-aks
- PostgreSQL: dify-postgres-9107e36a
- Vector Extension: pgvector

## Tags
#architecture #azure #kubernetes

    2. Upload to Dify:

    3. Click "Add Document"
    4. Upload test-architecture.md

    5. Wait for processing

    6. Verify Indexing:

    7. Check that document appears in the list
    8. Status should be "Completed"
    9. Click on document to see chunks

    Step 7: Test Retrieval¶

    1. Open Knowledge Base Testing:
    2. Click on the knowledge base

    3. Go to "Test" tab

    4. Test Queries: 

      Query 1: "What AKS cluster is used?"
Expected: Should return info about dify-aks

Query 2: "How is the vector database configured?"
Expected: Should return info about pgvector

Query 3: "What is the PostgreSQL host?"
Expected: Should return dify-postgres-9107e36a.postgres.database.azure.com

    5. Check Results:

    6. Verify that relevant chunks are returned
    7. Check similarity scores (should be > 0.70)
    8. Verify metadata is present

    Step 8: Create RAG Application¶

    Now create a chat application that uses this knowledge base:

    1. Create New App:
    2. Click "Studio" â†’ "+ Create Application"
    3. Name: Nirvana Knowledge Assistant

    4. Type: Chat App

    5. Configure App: 

      Model: gpt-4o-mini (Azure OpenAI)

    6. Add Knowledge Base:

    7. In "Context" section
    8. Click "+ Add Knowledge"
    9. Select "Nirvana Knowledge Portal"

    10. Retrieval Mode: Multi-path Retrieval

    11. System Prompt:

Your role is to help engineers with:
- Architecture decisions and ADRs
- Code examples and best practices
- Troubleshooting issues

RULES:

5. Use code examples when appropriate
6. Format responses in Markdown

CONTEXT:
The following documentation chunks are relevant to the user's question:
{context}

{query}

    12. Test the App:

    13. Click "Debug and Preview"
    14. Try the same test queries
    15. Verify responses are accurate and cite sources

    Step 9: Configure API Access¶

    1. Generate API Key:
    2. Go to App Settings
    3. Navigate to "API Keys"
    4. Click "+ Create API Key"
    5. Name: Knowledge Portal API

    6. Save the key securely

    7. Test API: 

      curl -X POST "http://10.0.2.91/v1/chat-messages" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": {},
    "query": "What is the AKS cluster name?",
    "user": "test-user",
    "response_mode": "blocking"
  }'

    Step 10: Enable WebApp (Optional)¶

    If you want a public-facing chat interface:

    1. Enable Site:
    2. Go to App Settings

    3. Toggle "Enable Site" to ON

    4. Get WebApp Code:

    5. Copy the WebApp code (e.g., abc123def456)

    6. URL will be: http://10.0.2.91/chatbot/abc123def456

    7. Update Knowledge Portal Config: 

      // apps/control-center-ui/.env.local
NEXT_PUBLIC_KNOWLEDGE_WEBAPP_CODE=abc123def456

    Verification Checklist¶

    • Knowledge base created and configured
    • Embedding model set to text-embedding-3-large
    • Retrieval settings configured (top_k=5, threshold=0.70)
    • Vector database connected (pgvector)
    • Test documents uploaded and indexed
    • Test queries return relevant results
    • RAG application created and tested
    • API key generated and tested
    • WebApp enabled (if needed)

    Troubleshooting¶

    Issue 1: "Connection to vector database failed"¶

    Solution: - Verify VPN is connected - Check PostgreSQL credentials - Test connection manually: 

    psql -h dify-postgres-9107e36a.postgres.database.azure.com \
     -U difyadmin -d nirvana_knowledge

    Issue 2: "Embedding generation failed"¶

    Solution: - Verify Azure OpenAI is configured in Dify - Check API key is valid - Verify model text-embedding-3-large is deployed

    Issue 3: "No results returned for queries"¶

    Solution: - Lower score threshold (try 0.60) - Increase top_k (try 10) - Check that documents were actually indexed - Verify vector column has data: 

    SELECT COUNT(*) FROM knowledge_chunks WHERE embedding IS NOT NULL;

    Issue 4: "Chunks are too large/small"¶

    Solution: - Adjust chunk size in settings - We use custom chunking, so this shouldn't affect our pipeline - Verify our chunks are ~800 tokens: 

    SELECT AVG(LENGTH(content)) FROM knowledge_chunks;
